　　第二天，沈柏寒在白板上写下了项目需求：

　　项目名称：噪音监测红绿灯

　　核心功能：
　　1. 实时监测环境分贝
　　2. 超过阈值（80分贝）时，红灯亮起+警报
　　3. 低于阈值时，绿灯亮起
　　4. 显示屏显示当前分贝值

　　技术栈：
　　- Arduino Uno（主控板）
　　- 声音传感器
　　- LED灯（红/绿）
　　- 蜂鸣器
　　- OLED显示屏

　　开发周期：7天

　　小宇看着这个清单，点点头："可行。"

　　"比如，声音传感器检测到80分贝，它就发送信号给Arduino，Arduino就让红灯亮起。"

　　小宇眼睛亮了起来："那就是一个if-else语句。"

　　"没错。"沈柏寒笑了，"硬件，其实也是代码的一种。"

　　接下来两天，他们在客厅的地板上，开始了"硬件马拉松"。

　　沈柏寒教小宇焊接电路板。

　　"焊接的时候要快，不然会烫坏元件。"他示范了一遍，"你看，这样——"

　　烙铁的尖端碰到焊锡，"滋"的一声，银色的液体流下来，把两个接点连接在一起。

　　小宇接过烙铁，小心翼翼地模仿。

　　第一次，焊点太大，像个疙瘩。

　　第二次，太小，根本没连上。

　　第三次，终于成功了——一个完美的圆锥形焊点。

　　"爸爸，你看！"小宇兴奋地举起电路板。

　　沈柏寒检查了一下："很好，合格。"

　　"继续。"

　　一整天，小宇就这样一个接点一个接点地焊。

　　焊到傍晚的时候，他的手指被烫了一小块，但他坚持把最后一个焊点也完成了。

　　陆以辰在旁边看着这一幕，心里涌起一种温暖。

　　这就是教育该有的样子——

　　不是灌输知识，而是一起创造。

　　不是评判分数，而是见证成长。

　　硬件组装完成后，就是写代码了。

　　沈柏寒打开电脑，创建了一个新的Arduino项目文件。

　　"今天我们要写的代码，逻辑很简单。"他说，"但要注意细节。"

　　他开始一行一行地敲代码，小宇在旁边看着：

```cpp
　　// 定义引脚
　　const int soundPin = A0;    // 声音传感器
　　const int redLED = 2;       // 红灯
　　const int greenLED = 3;     // 绿灯
　　const int buzzer = 4;       // 蜂鸣器

　　// 阈值
　　const int threshold = 80;

　　void setup() {
　　pinMode(redLED, OUTPUT);
　　pinMode(greenLED, OUTPUT);
　　pinMode(buzzer, OUTPUT);
　　Serial.begin(9600);
　　}

　　void loop() {
　　int soundLevel = analogRead(soundPin);
　　int decibel = map(soundLevel, 0, 1023, 30, 130);

　　if (decibel > threshold) {
　　digitalWrite(redLED, HIGH);
　　digitalWrite(greenLED, LOW);
　　tone(buzzer, 1000);
　　Serial.println("警告：噪音过大！");
　　} else {
　　digitalWrite(redLED, LOW);
　　digitalWrite(greenLED, HIGH);
　　noTone(buzzer);
　　}

　　delay(100);
　　}
```

　　"你能理解这段代码吗？"沈柏寒问。

　　小宇盯着屏幕，点点头："能。"

　　"那你说说看。"

　　"第一步，读取传感器的数值，范围是0到1023。"小宇说，"然后用map函数把它转换成分贝值，范围是30到130。"

　　"第二步，判断。如果分贝值大于80，就亮红灯，响蜂鸣器。否则，亮绿灯。"

　　"最后，延迟100毫秒，然后重复。"

　　"完全正确。"沈柏寒说，"那现在，你来优化这段代码。"

　　"优化？"

　　"嗯。"沈柏寒说，"这段代码有个问题——如果噪音持续超标，蜂鸣器会一直响，这样会很吵，反而成了新的噪音源。"

　　"你能想办法解决吗？"

　　小宇想了想："可以加一个计时器。如果噪音持续超标5秒，才响蜂鸣器。"

　　"很好。"沈柏寒说，"那你试试写出来。"

　　小宇接过键盘，开始敲代码。

　　他的速度不快，但很认真。

　　每写完一行，他都会停下来，确认逻辑没问题，然后继续。

　　二十分钟后，他完成了：

```cpp
　　unsigned long startTime = 0;
　　bool isNoisy = false;

　　void loop() {
　　int soundLevel = analogRead(soundPin);
　　int decibel = map(soundLevel, 0, 1023, 30, 130);

　　if (decibel > threshold) {
　　if (!isNoisy) {
　　startTime = millis();
　　isNoisy = true;
　　}

　　digitalWrite(redLED, HIGH);
　　digitalWrite(greenLED, LOW);

　　if (millis() - startTime > 5000) {
　　tone(buzzer, 1000);
　　}
　　} else {
　　digitalWrite(redLED, LOW);
　　digitalWrite(greenLED, HIGH);
　　noTone(buzzer);
　　isNoisy = false;
　　}

　　delay(100);
　　}
```

　　沈柏寒看了一遍，点点头："逻辑正确，代码简洁。非常好。"

　　"上传到Arduino，我们测试一下。"

　　小宇按下上传按钮，等待编译。

　　几秒钟后，Arduino上的灯开始闪烁——代码成功上传了。

　　这是他们的第一个作品。

　　朴素，简陋，但它能工作。

　　比赛时，小宇还需要在台上讲解自己的作品。

　　他拿出一张纸，写下了演讲稿的大纲：

　　小宇看着那张纸，皱起眉头。

　　"我要背这些吗？"

　　"不是背，是讲。"陆以辰说，"你只要说出你为什么做这个东西，它是怎么工作的。"

　　"但我不知道怎么说。"小宇说。

　　"那我们一点一点来。"陆以辰说，"首先，你为什么想做这个噪音监测器？"

　　小宇想了想："因为我害怕噪音。"

　　"很好。"陆以辰说，"那你继续说，为什么害怕？"

　　"因为……"小宇停住了。

　　他不知道该怎么描述那种感觉。

　　那种在嘈杂的环境里，感觉自己的CPU要烧毁的感觉。

　　"因为噪音让我感觉不舒服。"他最终说。

　　"那其他人呢？"陆以辰问，"有没有人也像你一样？"

　　小宇点点头："应该有。"

　　小宇愣了一下，然后慢慢点头。

　　他从来没这样想过。

　　他以为自己只是在解决自己的问题。

　　但陆爸爸说得对——如果他有这个问题，一定还有别人也有。

　　"所以，你的开场可以这样说：'我叫小宇，我对噪音很敏感。我做这个装置，是想帮助像我一样的人。'"

　　小宇重复了一遍："我叫小宇，我对噪音很敏感……"

　　声音很小，很不确定。

　　"别紧张。"陆以辰说，"你只是在讲述一个事实。"

　　但小宇就是记不住那些"尊敬的评委老师""感谢大家的聆听"之类的套话。

　　"陆爸爸，我能不能不说这些？"他问。

　　陆以辰想了想："可以。你就说你想说的。"

　　"但别人都会说这些。"小宇说，"我不说，会不会很奇怪？"

　　"也许会。"陆以辰说，"但重要的不是你说了什么客套话，而是你的作品能不能解决问题。"

　　"记住，你是个创客。创客不需要完美的演讲，只需要能工作的产品。"

　　小宇点点头，眼神坚定了一些。

　　比赛前一天晚上，小宇坐在床上，抱着那个噪音监测器。

　　电路板裸露在外面，电线歪歪扭扭，焊点大小不一。

　　和传单上那些光鲜亮丽的作品比起来，它看起来很简陋。

　　"爸爸。"小宇叫了一声。

　　沈柏寒走进来："怎么了？"

　　"我的作品……会不会太丑了？"

　　沈柏寒坐在床边，看着那个装置。

　　"你知道在程序员的世界里，有一句话吗？"他说。

　　"什么话？"

　　"'能跑就行。'"沈柏寒说，"意思是，代码不需要完美，只要能解决问题。"

　　"外表不重要，重要的是功能。"

　　"你的这个装置，它能准确地检测噪音，能及时地发出警报。"

　　"它做到了它应该做的事情。"

　　"那它就是成功的。"

　　小宇看着那个装置，慢慢点了点头。

　　"爸爸，明天我会紧张吗？"他问。

　　"会。"沈柏寒诚实地说，"我也会紧张。"

　　但也在传递着温度。

　　"作品也太简陋了吧……"

　　他想冲上去，把小宇抱下来，告诉他"没关系，我们回家"。

　　70分贝。

　　80分贝——

　　突然，台上小宇的装置亮了。

　　刺眼的红灯，像警报一样，瞬间吸引了所有人的注意。

　　紧接着，蜂鸣器开始响："嘀——嘀——嘀——"

　　全场瞬间安静了。

　　所有人都盯着那个装置。

　　红灯继续亮着，几秒钟后，因为环境安静了下来，它变成了绿灯。

　　小宇看着那个绿灯,突然找回了自己的语言。

　　"它亮了。"他说，声音很轻，但在安静的展厅里，清晰可闻。

　　"因为刚才太吵了。"

　　他指着那个装置："这是一个噪音监测器。"

　　"当环境分贝超过80时，红灯会亮起,提醒大家太吵了。"

　　"当分贝降下来，绿灯会亮起，表示安全。"

　　他停顿了一下，然后说：

　　"我做这个，是因为我对噪音很敏感。"

　　"在很吵的地方，我会感觉很难受。"

　　"但我不知道该怎么告诉别人'你们太吵了'。"

　　"所以我做了这个装置。"

　　"它可以帮我说出来——用红灯。"

　　台下，安静得能听见呼吸声。

　　小宇继续说："它的原理很简单。"

　　他指着电路板："这是声音传感器，它检测环境分贝。"

　　"这是Arduino，它是主控芯片，运行我写的代码。"

　　"代码逻辑就是一个if-else语句：如果分贝大于80，红灯亮；否则，绿灯亮。"

　　他拿出一张纸，上面手写着几行代码：

```
　　if (decibel > 80) {
　　light.red();
　　} else {
　　light.green();
　　}
```

　　"就这么简单。"他说。

　　然后他看向评委："我没有用很复杂的算法，也没有用云端。"

　　"因为这个问题不需要那些。"

　　"它只需要能工作。"

　　说完,他抱着装置，站在那里，等待评委的反应。

　　台下，沉默了几秒。

　　然后，掌声响了起来。

　　不是礼貌性的掌声，而是真诚的、热烈的掌声。

　　一个评委站了起来，是个五十多岁的工程师。

　　"小朋友，我能问你一个问题吗？"

　　小宇点点头。

　　"这个装置，从设计到制作，都是你自己完成的吗？"

　　"焊接是我爸爸教我的。"小宇诚实地说，"但代码是我自己写的。"

　　"很好。"那个评委说，"那你能现场演示一下吗？"

　　小宇点头。

　　他把装置放在桌上，然后说："请大家一起鼓掌。"

　　台下的人愣了一下,然后开始鼓掌。

　　掌声越来越大——

　　红灯亮了。

　　蜂鸣器响了。

　　掌声停止——

　　绿灯亮了。

　　蜂鸣器停了。

　　"完美。"那个评委笑了,"这才是真正的创客作品。"

　　"它不需要华丽的外壳,不需要复杂的算法。"

　　"它只需要解决一个真实的问题。"

　　"谢谢你，小朋友。你做得很好。"

　　小宇鞠了个躬，抱着装置走下台。

　　台下，沈柏寒和陆以辰已经等在那里。

　　沈柏寒一把抱住小宇："你太棒了！"

　　小宇把脸埋在沈柏寒怀里,身体在颤抖。

　　不是害怕，是激动。

　　"爸爸，"他的声音有些哽咽,"我做到了。"

　　"是的，"沈柏寒说，"你做到了。"

　　"而且你做得比任何人都好。"

　　下午四点，颁奖典礼开始。

　　主持人宣布获奖名单：

　　"金奖：5号选手，《基于深度学习的智能分类系统》"

　　"银奖：12号选手，《物联网智能家居控制中心》"

　　"铜奖……"

　　小宇没有获奖。

　　他坐在座位上,看着那些获奖的孩子上台，脸上没有失落，只是平静。

　　"31号选手，沈小宇。"主持人突然叫了他的名字。

　　小宇愣了一下。

　　"虽然没有获得金银铜奖，但评委会一致认为，你的作品体现了真正的创客精神——**发现问题，解决问题**。"

　　"因此，我们决定授予你**'最佳创意奖'**。"

　　"请上台领奖。"

　　小宇站起来,走上台。

　　评委把一个奖杯递给他——比他的脸还大。

　　"小朋友，"那个工程师评委说，"你的作品也许不是最复杂的，但它是最诚实的。"

　　"希望你保持这种精神。"

　　"不要为了迎合评委而做作品，要为了解决问题而创造。"

　　"这才是工程师该有的样子。"

　　小宇抱着奖杯，用力点头。

　　走出科技馆的时候，夕阳正好。

　　小宇抱着那个比他脸还大的奖杯，奖杯在阳光下反射着光。

　　"小宇，感觉怎么样?"沈柏寒问。

　　小宇想了想："还可以。"

　　"只是还可以？"陆以辰笑了。

　　"嗯。"小宇说，"因为我发现，那个工程师评委的眼镜是不对称的。左边的镜片比右边厚0.2毫米。"

　　沈柏寒和陆以辰对视一眼,然后大笑起来。

　　这就是小宇。

　　永远在观察，永远在分析，永远在用他的方式理解世界。

　　"对了，爸爸。"小宇突然说。

　　"嗯？"

　　"刚才你在台下鼓掌，是故意的，对吗？"

　　沈柏寒愣了一下，然后笑了："你发现了？"

　　"嗯。"小宇说，"因为你鼓掌的频率和力度都超出了正常范围。"

　　"你是想触发我的装置。"

　　"聪明。"沈柏寒揉了揉他的头,"我知道你在台上说不出话，所以我想，也许可以让你的作品替你说话。"

　　"谢谢爸爸。"小宇说。

　　"不客气。"沈柏寒说，"这是我们的默契。"

　　他们走在回家的路上。

　　小宇抱着奖杯，沈柏寒提着那个朴素的噪音监测器,陆以辰走在旁边。

　　三个人的影子，在夕阳下拉得很长。

　　"爸爸。"小宇又叫了一声。

　　"嗯？"

　　"我以后还能参加这种比赛吗？"

　　"当然。"沈柏寒说，"只要你想。"

　　"那下次,我想做一个更难的。"小宇说，"我想做一个……"

　　他想了想："一个能检测情绪的装置。"

　　"为什么？"

　　"因为我不太会看别人的表情。"小宇说，"如果有个装置能告诉我，'这个人现在是高兴还是生气'，我就能更好地和人相处了。"

　　沈柏寒和陆以辰又一次对视。

　　这个孩子，总是在用他的方式，试图理解这个复杂的世界。

　　"好。"沈柏寒说，"那我们下次就做这个。"

　　"不过，"陆以辰说，"在做情绪检测器之前，也许我们可以先学习一下，人是怎么表达情绪的。"

　　"比如，当一个人笑的时候，他可能是开心。"

　　"当一个人皱眉的时候，他可能是困惑或者生气。"

　　小宇点点头："这个我可以学。"

　　"那我们回家就开始。"陆以辰说。

　　三个人继续往前走。

　　路过一家便利店的时候，沈柏寒突然停下来。

　　"要不要庆祝一下？"他说，"买点冰淇淋？"

　　小宇的眼睛亮了起来："要！"

　　他们走进便利店，每人选了一支冰淇淋。

　　走出来的时候，小宇一边吃一边问："爸爸,你觉得今天那些获奖的作品,真的比我的好吗？"

　　沈柏寒想了想："也许在评委的标准里,它们更符合'科技创新'的定义。"

　　"但在我看来，"他蹲下来，看着小宇的眼睛,"你的作品是最真实的。"

　　"因为它解决的,是你自己的问题。"

　　"而那些获奖的作品,很多都是为了迎合评委，为了拿奖而做的。"

　　"它们也许很炫，但不一定有用。"

　　"而你的作品,虽然简陋,但它真的能工作。"

　　"这才是工程师的骄傲。"

　　小宇点点头，然后说："爸爸,你以前也是这样的吗？"

　　"什么？"

　　"就是……"小宇想了想该怎么表达,"就是在很多人都在做华丽的东西的时候,你还在坚持做真正有用的东西？"

　　沈柏寒愣住了。

　　他想起了很多年前,在互联网泡沫的时代。

　　所有人都在追逐概念,追逐融资,追逐上市。

　　而他，只是想写出一行完美的代码。

　　"是的。"他最终说，"我试过。"

　　"但很多时候,我失败了。"

　　"因为这个世界，往往更看重表面,而不是内核。"

　　"那为什么你还要教我这样做？"小宇问。

　　沈柏寒看着他，突然笑了。

　　"因为我希望你不要像我一样。"他说。

　　"我希望你能一直坚持自己认为对的东西。"

　　"哪怕全世界都在做华丽的PPT,你还是要写真正能工作的代码。"

　　"哪怕所有人都在追逐金奖,你还是要解决真实的问题。"

　　"这才是创客精神。"

　　"这才是工程师的尊严。"

　　小宇听着，慢慢点头。

　　"我记住了，爸爸。"他说。

　　"我会一直这样做的。"

　　晚上，小宇睡了。

　　沈柏寒坐在书桌前,打开笔记本。

　　【寄养日志·Day 18】

　　今天,小宇参加了他人生中第一次比赛。

　　科技创新大赛。

　　我本来很担心,怕他会像上次幼儿园面试那样,被那些标准化的要求打击。

　　但这次不一样。

　　因为这次,他展示的是自己真正的才华。

　　他做了一个噪音监测器。

　　很简陋,电路板裸露在外面，焊点大小不一。

　　当他在台上说不出话的时候,我制造了噪音,触发了他的装置。

　　红灯亮起的那一刻，我看见所有人都愣住了。

　　然后他们明白了——

　　这不是一个为了比赛而做的展示品。

　　这是一个真正能解决问题的产品。

　　评委说,虽然没有获得金奖,但他的作品体现了真正的创客精神。

　　他获得了"最佳创意奖"。

　　但在我心里,他的作品是最真实的。

　　"坚持住。"

　　"你的代码，会被人看见的。"

　　"你的初心,不会白费的。"

　　但至少现在,我在小宇身上,看见了那份坚持的延续。

　　沈柏寒合上笔记本,走到小宇的房间门口。

　　门虚掩着，他轻轻推开。

　　小宇睡得很沉，怀里抱着那个奖杯。

　　月光洒在他脸上，那张稚嫩的脸上,带着满足的笑容。

　　沈柏寒站在门口,看了很久。

　　然后他轻轻关上门,走回客厅。

　　"小宇睡了？"他问。

　　"是啊。"沈柏寒说,"我没想到他能做到。"

　　"你也很棒。"陆以辰说,"在台下制造噪音那一下,很聪明。"

　　"那是我唯一能想到的办法。"沈柏寒说,"我知道他的装置能说话,所以我让它替他说话。"
