# 两种爱的语言 - 文档结构说明

## 概述

本项目是基于 Fumadocs 构建的文档网站，用于展示小说《两种爱的语言》的内容。

## 目录结构

### 主线章程（章节 1-61）
`/docs/mainline/`

**时间线**：2024年12月 - 2026年8月

**主要内容**：
- 第1-10章：初识与重启（2015年夏天初识，2024年重逢）
- 第11-20章：深渊与挣扎（陆以辰的抑郁症困境）
- 第21-30章：真相与审判（HIV确诊，关系确立，面对外界审视）
- 第31-40章：日常与反思（日常生活、职业变迁、时代背景）
- 第41-50章：孤独与情感（疫情记忆、情感深化）
- 第51-61章：系统与未来（产品开发、创业、未来展望）

### 回溯卷：时代洪流
`/docs/timeline/`

**时间线**：2015-2024

编年史形式展现十年社会变迁，采用双视角交替叙述：
- 章-0：2015年夏·上海弄堂里的光（陆视角：初次相遇）
- 章-1：象牙塔里的异乡人（陆视角：大学生活）
- 章-2：新婚之夜的虚假和平（沈视角：婚姻开始）
- 章-3：第一次试错式恋爱（陆视角：初恋）
- 章-4：无性婚姻的确立（沈视角：婚姻困境）
- 章-5：海德格尔的召唤（陆视角：哲学追寻）
- 章-6：深夜的交友软件（沈视角：身份探索）
- 章-7：疫情中的封闭（陆视角：2020年疫情）
- 章-8：离婚前夜（沈视角：婚姻结束）
- 章-9：靠近的渐行线（陆视角：逐渐接近）
- 章-10：系统的重置归零（沈视角：生活重启）
- 章-11：平行线（双视角：命运交汇）
- 章-12：濒死之光（沈视角：危机时刻）

**叙事策略**：回溯卷采用"陆"、"沈"双视角交替，通过各自的视角回顾2015-2024年间的关键时刻，补充主线叙事中的历史背景。

### 附录
`/docs/appendix/`

- `characters.mdx` - 角色设定
- `timeline-reference.mdx` - 完整时间线参考

## 推荐阅读顺序

### 方案一：线性阅读（推荐首次阅读）
1. **主线章程**（第1-61章）- 按章节顺序阅读，体验完整的主线故事
2. **回溯卷**（章-0至章-12）- 阅读完主线后，回顾两位主角的十年历程

这种阅读方式适合：
- 首次阅读的读者
- 喜欢先了解主要故事情节的读者
- 希望带着对人物的好奇心去探索背景故事

### 方案二：编年体阅读（适合重读）
1. **回溯卷**（章-0至章-12）- 先了解2015-2024年的历史背景
2. **主线章程**（第1-61章）- 再阅读2024-2026年的主线故事

这种阅读方式适合：
- 重读或深度分析的读者
- 喜欢按时间线理解故事的读者
- 希望完整把握人物成长弧线的读者

### 方案三：交织阅读（进阶）
主线与回溯卷交织阅读，在关键节点穿插历史回顾：
- 主线第1-10章 → 回溯章-0至章-2（初识背景）
- 主线第11-30章 → 回溯章-3至章-8（关系建立期的历史）
- 主线第31-61章 → 回溯章-9至章-12（当下与过去的对话）

这种阅读方式适合：
- 有时间和耐心的深度读者
- 喜欢在现实与回忆间穿梭的叙事体验
- 希望同步理解现在与过去关联的读者

## 核心主题

1. **非对称的爱**：年龄差（16岁）、健康差异、思维方式的差异
2. **时代的幸存者**：从"双创"到疫情，从P2P到AI，两个普通人的求生
3. **具体的胜利**：对抗宏大理论，捍卫"此时此地"的真实经验

## 技术栈

- **框架**：Next.js 16.1.1
- **文档系统**：Fumadocs (fumadocs-core 16.4.7)
- **内容格式**：MDX
- **样式**：Tailwind CSS 4.1.18

## 本地开发

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 访问文档
# http://localhost:3000/docs
```

## 内容格式

所有章节采用 MDX 格式，包含 YAML frontmatter：

```mdx
---
title: "章节标题"
---

章节内容...
```

## 导航结构

文档使用 `meta.json` 文件控制导航顺序和结构：

- `content/docs/meta.json` - 根导航
- `content/docs/mainline/meta.json` - 主线章节列表（第1-61章）
- `content/docs/timeline/meta.json` - 回溯卷章节列表（章-0至章-12）
- `content/docs/appendix/meta.json` - 附录内容列表
